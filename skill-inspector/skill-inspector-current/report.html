概述

Overview

一次分析一个 skill 来源，并产出结构化的 `report.json` 与整理好的 `report.html`。 这是一个 agent-native 版本，明确要求使用当前目录里的模板、schema 与 prompt，而不是依赖 Python 分析脚本。

Analyze one skill source at a time and produce: This version is agent-native. Do not rely on Python analysis scripts. Use the templates and schema in this directory and let the current agent perform the analysis.

结构化输出优先

Structured JSON First

在生成 HTML 之前，必须先创建符合 `templates/report.schema.json` 的 `report.json`，并把它当成唯一 canonical artifact。 这份 skill 现在明确禁止生成自定义中间 report shape 或手写 Mermaid 作为主路径，所有渲染都应从标准 JSON 出发。

Before generating HTML, create a `report.json` object that conforms to: `templates/report.schema.json`. `report.json` must be the first valid canonical artifact. Do not create a custom intermediate report shape and convert it later.

翻译质量控制

Translation Quality Controls

非中文 skill 默认走 `full`，并要求 section 与 row 与英文骨架一一对应；缺行、空 `zh`、占位文本和整句英文复制都属于失败。 `prompts/translation.md` 还补了正反样例，覆盖标题、纯 prose、路径、命令、变量以及典型作弊样本，用来对齐模型风格而不是只靠 blacklist。

Default to `full` for non-Chinese skills. For `full` output, preserve section count, section order, and row count exactly; do not delete, merge, compress, or skip rows. Load: `prompts/translation.md`.

Workflow 建图策略

Workflow Guidance

workflow 现在强调结构化 `nodes` / `edges`，并要求渲染器从这些结构推导 Mermaid，而不是直接手写图文本。 `prompts/workflow-generation.md` 新增了 `phase_serial` 与 `phase_with_internal_steps` 等模式指引，目的不是更强约束，而是让模型先识别流程类型，再选择合适图法。

Use structured workflow data as the canonical representation. Let the local renderer derive Mermaid from the workflow structure. Load `prompts/workflow-generation.md` when the workflow graph needs language-level interpretation rather than simple heuristic extraction.

渲染与阅读体验

Rendering and Reading Experience

HTML 只能从标准 `report.json` 渲染，且在 Mermaid 前会把 workflow id 映射成安全节点 id，避免 `style`、`class` 等保留字破坏图。 渲染层现在保留逐行真源做校验，同时把连续 prose 行聚合成段落、把列表行聚合成列表块，让双语阅读更接近真实文档。

When rendering Mermaid, never pass raw workflow ids through directly; always map them to Mermaid-safe ids first. The local renderer should fill the HTML template from `report.json`.

安全评估

低风险
这份 skill 的主风险集中在本地文件读写、远程 raw 链接读取和客户端 HTML/JSON 渲染，而不是危险系统操作。当前版本已经把 canonical JSON、Mermaid-safe id 和假完整翻译拦截写进主流程，整体风险较低。
local-file-write low
默认会在本地输出 `normalized-source.json`、`report.draft.json`、`report.overlay.template.json`、`report.json` 和 `report.html`，副作用局限在报告产物目录。
证据: Write: `%当前目录%/skill-inspector/%skill-name%/report.json` and `%当前目录%/skill-inspector/%skill-name%/report.html`.
remote-source-fetch medium
支持远程 skill 链接，并会先尝试 raw URL，再回退到原 GitHub 页面；风险主要在外部来源可信度和网络可达性。
证据: Fetch the raw URL first when the source is a GitHub file link. Fall back to the original GitHub page only when the raw URL returns a non-success response, empty body, or HTML page content.
html-json-rendering low
页面会内嵌 Mermaid 和 JSON payload，但当前规则已经要求 JSON.parse-compatible 嵌入与 Mermaid-safe id 映射，降低了脚本上下文和保留字冲突风险。
证据: When rendering Mermaid, never pass raw workflow ids through directly; always map them to Mermaid-safe ids first.

优化建议

为 self-report 固定 canonical fixture
把当前 `SKILL.md` 的自分析报告做成受版本控制的固定 fixture，并把 gh-pages 发布直接绑定到这份 canonical 产物，减少线上报告滞后。
优先级: high
给 workflow 引入 phase 容器渲染
当前 prompt 已能区分 `phase_serial` 与 `phase_with_internal_steps`，下一步值得在渲染层提供 phase 容器或泳道，让 phase 主干与内部 step 的层级更直观。
优先级: medium
把段落级展示升级成显式 block 数据
现在段落聚合发生在 render 阶段。后续可以考虑给 translation 增加 block-level presentation metadata，让 JSON 本身也承载阅读层语义。
优先级: medium
