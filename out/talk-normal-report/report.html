概述

`talk-normal` 是一个用于压制 AI agent 套话和“企业风废话”的安装型 skill，主要面向中英双语场景。
它并不是每轮手动调用的 workflow skill，而是一次性把行为规则安装进工作区，让后续回复默认继承这些约束。
talk-normal is an installer-style skill for suppressing filler, corporate LLM output, with bilingual support.
It is not a per-turn workflow skill. It installs a persistent behavior modifier into the workspace.

它具体做什么

执行后，它会把 `prompt.md` 的内容注入当前工作区的 `AGENTS.md`，并用显式的 BEGIN/END marker 包裹规则块。
因此它更像一个“行为安装器”，而不是一个逐步执行的分析流程。
It injects the contents of `prompt.md` into the workspace `AGENTS.md`, wrapped by explicit BEGIN/END markers.
This makes it closer to a behavior installer than a turn-by-turn workflow process.

如何运行

安装或更新时运行 `bash install.sh`；卸载时运行 `bash install.sh --uninstall`。
脚本是幂等的：重复执行会先移除旧 block，再写入最新规则。
Run `bash install.sh` to install or update, and `bash install.sh --uninstall` to remove.
The script is idempotent: it strips the old block before writing the latest rules.

规则内容

`prompt.md` 里的规则主要用来压缩几类常见废话模式：开场套话、总结式收尾、平衡作文式比较、条件式后续菜单等。
其中一个关键要求，是直接给正向结论，避免“不是 X，而是 Y”这种修辞框架。
The rules in `prompt.md` target common filler patterns such as openings, summary endings, balanced comparisons, and conditional follow-up menus.
A core rule is to prefer direct positive conclusions over reject-then-correct phrasing.

更新、兼容性与验证

更新路径是先 `clawhub update talk-normal`，再重新执行安装脚本。兼容性目标是任何支持 system prompt 或 custom instruction 的模型。
`TEST_RESULTS.md` 提供 benchmark 数据，用来支撑压缩率和效果声明。
Updating means `clawhub update talk-normal`, then rerunning the installer. Compatibility is broad for models that honor system prompts or custom instructions.
`TEST_RESULTS.md` contains benchmark evidence backing the reduction claims.

贡献方式

`CONTRIBUTING.md` 要求新增规则必须基于真实输出中的真实废话模式，而不是纯个人偏好。
`CONTRIBUTING.md` requires each new rule to be grounded in a real slop pattern from real output, not just personal taste.

安全评估

中风险
中风险表示它会执行 shell 脚本并修改工作区 `AGENTS.md`，但作用范围主要局限于本地文本规则注入，没有直接涉及高权限系统设置。
shell-installer 中风险
通过 `bash install.sh` 执行本地脚本，存在脚本执行风险，需要审阅脚本内容。 证据：bash install.sh
workspace-rule-injection 中风险
会修改或创建 `AGENTS.md`，属于持久化行为变化，需要明确影响范围和回滚方式。 证据：The contents of `prompt.md` get injected into your `AGENTS.md`.

优化建议

补一段安装后验证步骤建议在 `How to run` 或 `What gets installed` 后面补一句“安装后如何确认已生效”，例如检查 `AGENTS.md` 中的 marker block 是否存在。
补一段 uninstall 影响说明建议明确 uninstall 只删除 marker 包裹的规则块，不会影响其他手写规则，以减少用户对回滚风险的顾虑。
压缩 benchmark 摘要的表达当前 benchmark 信息已经很强，建议把摘要区的效果表达再压短一点，避免和 `TEST_RESULTS.md` 的详表形成重复。
