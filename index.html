概述

Overview

这份 skill 用来分析单个 skill 来源，并产出结构化 `report.json` 与可读 `report.html`。
当前版本采用 agent-native 流程，判断型内容通过 overlay 补充，结构和展示由仓库内模板与 schema 约束。
Analyze one skill source at a time and produce a structured `report.json` and a polished `report.html`.
This version is agent-native. Use the templates and schema in this directory and let the current agent perform the analysis.

输入

Inputs

支持三类输入：贴入的 skill 全文、本地 skill 文件路径、远程 skill 链接。
Supported inputs: pasted skill text, local path to a skill file, and remote skill link.

主工作流

Required Workflow

第一步读取并归一化来源，提取 title、frontmatter、sections、commands、references 和 workflow signals，并生成紧凑工作集与 `reportSeeds`。
第二步先生成符合 `templates/report.schema.json` 的 `report.json` 结构，workflow 以 `nodes` 和 `edges` 为标准表示，`install` 作为确定性本机目录检查字段。
第三步用 `templates/report.html` 渲染 HTML，workflow 是主区块，translation 是次区块，meta 区块保持紧凑。
第四步写出 `out/report.json` 与 `out/report.html`，可选链路还包括 `normalize-skill`、`build-report-draft`、`build-report-overlay-template`、`finalize-report` 四段预处理。
Step 1 reads and normalizes the source, extracting title, frontmatter, sections, commands, references, and workflow signals into a compact working set with `reportSeeds`.
Step 2 builds a schema-shaped `report.json`, treats workflow graph data as the canonical representation, and keeps `install` as a deterministic local-directory check.
Step 3 fills `templates/report.html`, keeping workflow as the primary section, translation as the secondary section, and meta sections compact.
Step 4 writes `out/report.json` and `out/report.html`, with optional helper stages for normalization, draft generation, overlay templating, and finalization.

翻译与洞察规则

Translation and Insight Rules

翻译层加载 `prompts/translation.md`，中文要求简洁、准确、可读，commands、paths、URLs、variables 和 frontmatter keys 保持英文。
洞察层加载 `prompts/insights.md`，重点产出简短具体的 suggestions，并围绕结构、引用、安全边界和可维护性给出判断。
Translation loads `prompts/translation.md`, keeps Chinese concise and readable, and preserves commands, paths, URLs, variables, and frontmatter keys in English.
Insights load `prompts/insights.md`, generate short concrete suggestions, and evaluate structure, references, safety boundaries, and maintainability.

输出契约

Output Constraints

`report.json` 是唯一规范分析输出，HTML 必须忠实反映 JSON。
输出层要求引用去重、证据精简、至少一条具体建议，以及始终提供 `install` 结果。
内嵌 `<script type="application/json">` 的 JSON 需要保持 `JSON.parse` 兼容，并处理脚本上下文转义。
`report.json` is the canonical analysis output and HTML must reflect it faithfully.
The output contract requires deduplicated references, compact evidence, at least one concrete suggestion, and an `install` result in every run.
Embedded `<script type="application/json">` payloads must stay `JSON.parse` compatible and escape script-context hazards correctly.
