概述

一次分析一个 skill 来源，并生成结构化的 `report.json` 与整理好的 `report.html`。
这是一个 agent-native 版本，不依赖 Python 分析脚本，而是要求当前 agent 按模板和 schema 完成分析。
Analyze one skill source at a time and produce a structured `report.json` and a polished `report.html`.
This version is agent-native. Do not rely on Python analysis scripts. Use the templates and schema in this directory and let the current agent perform the analysis.

输入

支持的输入包括：
- 粘贴的 skill 全文
- 本地 skill 文件路径
- 远程 skill 链接
Supported inputs:
- pasted skill text
- local path to a skill file
- remote skill link

必经流程

先读取并归一化目标 skill，抽出标题、frontmatter、section、命令、引用和 workflow 线索。
不要立刻生成最终报告。
Read and normalize the source first. Extract title, frontmatter, sections, commands, references, and workflow signals.
Do not generate the final report yet.

- `translation`

翻译规则

需要读取 `prompts/translation.md`。
- 中文应当简洁、准确、可读
- 产品名、框架名、工具名、命令、路径、URL、变量名和 frontmatter key 保持英文
- 不要总结
- 不要额外解释
Load `prompts/translation.md`.
- Chinese should be concise, accurate, and readable
- Product names, framework names, tool names, commands, paths, URLs, variables, and frontmatter keys stay in English
- Do not summarize
- Do not add explanation

建议规则

需要读取 `prompts/insights.md`。
- 建议必须简短、具体、可执行
- 重点关注结构、引用、安全边界和可维护性
- 避免泛泛而谈
引用附录以 `summary` 为主说明，`condition` 主要放到 workflow / Mermaid 层；如果引用有触发条件，优先放到 Mermaid 边标签里，而不是在附录正文里重复。
Load `prompts/insights.md`.
- Suggestions must be concrete and short
- Focus on structure, references, safety boundaries, and maintainability
- Avoid generic advice
Use `summary` as the primary appendix description for each reference. Keep `condition` primarily in the workflow / Mermaid layer. When a reference has a trigger condition, prefer showing it on Mermaid edges instead of repeating it in appendix prose.

输出约束

必须先生成 JSON，再生成 HTML；HTML 必须反映 JSON，而不是另起一套解释。
不要发明源文件里没有支持的 section；不要翻译命令、代码、路径、URL 或 frontmatter key。
- 引用需要去重，并保留最有信息量的 evidence line
- 必须始终给出至少一条具体建议
- 必须始终提供 `install` 结果，即使它是启发式或 `unknown`
Generate JSON first. HTML must reflect the JSON, not a separate interpretation.
Do not invent sections that are unsupported by the source. Do not translate commands, code, paths, URLs, or frontmatter keys.
- Deduplicate repeated references while preserving the most useful evidence line
- Always provide at least one concrete suggestion
- Always provide an `install` result, even when it is heuristic or `unknown`

安全评估

低风险
当前输出主要受固定 schema 和 HTML 模板约束，暂未识别出明显高危操作或高权限依赖。
template-controlled-output 低风险
输出结构受模板和 schema 约束，内容漂移风险较低。 证据：The workflow constrains output through a fixed schema and HTML template.

优化建议

补充 install 判定说明建议在 `SKILL.md` 或 schema 示例中明确 install 结果如何给出 `installed / not_found / unknown`，减少不同 agent 的自由发挥。
增加占位约定说明建议补一段模板占位约定，明确 `translation_sections_html`、`references_html`、`install_html` 这类区域由谁、以什么格式填充。
给出去重示例建议在 examples 中加入“同一引用在不同语境下去重后如何保留最有信息量的 evidence line”的示例。
